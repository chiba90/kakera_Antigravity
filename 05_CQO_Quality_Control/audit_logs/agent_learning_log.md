# AI Agent Learning & Update Log

このファイルは、ユーザーからの指摘事項（フィードバック）や、システム・ワークフローへの変更履歴を記録し、同じミスを繰り返さないための自律的学習ログです。
AIエージェントのタスク着手前やコード修正前にこのログを参照し、過去の失敗と改善措置を必ず確認してください。

## 📥 読み込み（Read）と 📤 記録（Write）のタイミング
AIは以下のタイミングを絶対のトリガーとして、本ファイルを運用すること。

**【📥 読み込み（参照）のタイミング：作業の「前」】**
1. ユーザーから新たなタスク（執筆、リライト、スクリプト作成など）を依頼された直後。
2. 既存のルールやワークフロー、アーキテクチャ定義を変更・修正しようとする前。

**【📤 書き込み（記録）のタイミング：事象の「直後」】**
1. ユーザーから明確な「ダメ出し」や「修正指示（ここはこうして等）」を受け、その修正対応が完了した直後。
2. AI自身の提案や出力がエラー・仕様不適合を起こし、その根本原因と解決策が判明した直後。
3. ルールファイルやワークフロー（`.agent/rules/` や `.agent/workflows/` 等）の構造を変更・アップデートしたセッションの終端。
- ※ 追記時は、必ず**降順（最新の記録が一番上）**になるように記載すること。

## [2026-05-30] 【品質改善】記事 #1836（い草ラグ）振り返りからの学び
- **課題①: 日本語原稿内への英語「of」混入のすり抜け**
  - **事象**: 記事 #1836 の 425行目において `伝統が持つ本来 of 精神性や` という英語「of」がすり抜けて公開されていた。
  - **原因**: 執筆および自動品質監査スクリプト `validate_headings.ps1` の実行タイミングがWordPress入稿プロセスと完全に同期していなかったか、あるいは自動監査の結果を見落としていた可能性。
  - **対策**: 今後、CCOおよびCQOは、WordPressへ入稿される最終データとローカルHTMLの完全同期を厳密にし、自動監査スクリプト `validate_headings.ps1` を強制実行して「本文全体の ` of ` 混入」を漏れなく検出し、1点でも混入があれば進行を完全ブロック（Exit 1）するルールを死守する。
- **課題②: JSON-LDの画像URLと本文内画像のパス乖離**
  - **事象**: 本文内画像が `wp-content/uploads/2026/05/` であったのに対し、JSON-LD内のURLが `wp-content/uploads/final_assets/` のままであり、構造化データの画像パスが乖離していた。
  - **対策**: 次回以降、画像の配置先URLがWordPress入稿時に変更される場合は、JSON-LD内の画像URLもCCOフェーズで完全に同期・置換した状態で納品する。
- **課題③: 最終章におけるH3階層化の未達成**
  - **事象**: 最終章のH2「タイパの砂漠に『余白』を〜」の配下にH3見出しが配置されておらず、のっぺり化を防ぐための「全H2下にH3配置」の基準が部分的に緩んでいた。
  - **対策**: 今後CMO（戦略）およびCCO（執筆）は、全てのH2に対して漏れなくH3を1つ以上配置し、論理的な階層化を徹底する。

*(普遍的なルールは `executive_summary.md` へ移管され、古いログは `archive/` へ正常に退避・ガベージコレクションされました)*
